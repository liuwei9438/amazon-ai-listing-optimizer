# Amazon AI Listing Optimizer V2.0 Core P1.4

## 本版重点

1. **重复产品缓存**
   - 使用标题、五点、详情和变体信息生成产品指纹。
   - 相同产品的产品理解结果只调用一次 AI。
   - 相同产品 + 相同语言的完整文案优先读取缓存。
   - 不同 SKU 仍保留为独立表格行。

2. **任务检查点**
   - 每完成 5 条结果自动保存一次。
   - 浏览器刷新后，页面顶部会出现“恢复上次优化结果”。
   - 失败项重试或图片重试后也会保存最新完整表格。
   - 可点击“立即保存当前任务”。

3. **失败池继续收敛**
   - 每次只处理当前失败项。
   - 成功项立即移出失败池。
   - 重试后写回同一份完整结果，最后统一导出。

4. **速度优化方式**
   - 本版优先实现安全的重复产品去重和缓存命中。
   - `core/batch_processor.py` 提供重复产品分组能力。
   - 本版没有把 20 个不同产品强行塞进同一次 AI 请求，避免单个 JSON 错误导致整批失败。

## 完整替换

建议在 `v2.0-core-test` 分支完整替换：

- `app.py`
- `requirements.txt`
- `core/`
- `storage/`
- `prompts/`

不要上传：

- `__pycache__/`
- `*.pyc`

## Streamlit Cloud 说明

当前检查点和缓存保存在 Streamlit 运行容器的本地目录中：

- 普通页面刷新、脚本 rerun 通常可以恢复。
- 应用重新部署、容器迁移、休眠后重新创建实例时，本地文件可能被清除。
- 需要跨部署永久保存时，下一阶段应将任务和缓存迁移到 Supabase、R2、S3 或数据库。

## 验证建议

先用 20–50 个产品测试：

- 中途刷新后是否能恢复已保存结果。
- 重复产品第二次是否显示缓存命中。
- 失败项重试后数量是否逐轮减少。
- 图片、Cloudinary、多语言、SKU 和 Excel 格式是否保持正常。
